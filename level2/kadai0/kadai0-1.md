## サービス 1: Slack

### サービスの概要

- チーム向けのビジネスチャットツール
- チャット形式でのコミュニケーション、ファイル共有、オンライン会議ツールなどと連携して仕事を効率化できる
- 現在は複数の技術により構築されているが、初期開発段階では PHP がサーバーサイドで使用されていた

### 実装されている機能

1. チャンネル機能
   - プロジェクトやチームごとに会話の場所を分け、メンバーで共有できる
2. ダイレクトメッセージ
   - 1 対 1 や少人数の会話ができる
3. ファイル共有
   - 画像・PDF・ZIP・Excel などのアップロード、共有ができる
4. メンション・通知機能
   - 必要な人に確実にメッセージを届けられる
5. 外部サービス連携
   - Google Drive、Zoom、GitHub、Notion など数千のサービスと連携できる
6. 検索機能
   - 過去の会話やファイルをキーワードで検索できる
7. Bot 機能 / API 連携
   - ChatGPT 連携など、自動返信や情報取得などの自動化ができる

## サービス 2: Facebook

### サービスの概要

- 世界的に有名な SNS（ソーシャル・ネットワーキング・サービス）
- 多数のユーザーがリアルタイムで投稿、コメント、メッセージ、プロフィール管理などを行う

### 実装されている主な機能

1. 投稿・タイムライン表示、いいね、コメント機能
   - ユーザーの投稿をタイムライン形式で表示し、リアクション（いいね）やコメントを行える
2. 友達／フォロー機能、プロフィール管理
   - 自分とつながりたいユーザーに友達申請ができ、つながりの範囲を管理できる
3. メッセージ／チャット機能、通知機能
   - 1 対 1 または複数人でチャットができ、新しいメッセージやイベントを通知する
4. 画像・動画のアップロード、アルバム管理
   - 思い出をアルバムとして整理でき、家族や友達と共有可能
5. グループ機能、ページ機能、イベント管理機能
   - 趣味やコミュニティごとにグループを作成でき、イベントの開催も可能

## サービス 3: CAMPFIRE

### サービスの概要

- 日本を代表するクラウドファンディングサービス
- クリエイターやプロジェクトオーナーが資金調達のためにプロジェクトを掲載し、支援者が支援を行うことができるプラットフォーム

### 実装されている主な機能

1. プロジェクト一覧の表示、検索、カテゴリー絞り込み、フィルタ機能
   - 多数のプロジェクトをカテゴリーや人気順で整理し、目的に合うものを探しやすくする
2. プロジェクト詳細ページ、リターン内容の表示、支援機能
   - プロジェクトの目的や進捗内容を閲覧でき、支援金額に応じたリターンを選べる
3. ユーザー登録・ログイン機能、マイページ機能
   - 支援履歴の確認やプロフィール情報の管理が可能
4. 決済機能、支援履歴管理、コメント／更新通知機能
   - クレジットカードなどによる決済が行え、活動報告を受け取ることができる
5. プロジェクトオーナー向け管理機能
   - 支援者へのメッセージ配信や進捗レポートの共有など、運営に必要な機能が使える

### 技術用語

1. API（Application Programming Interface）

- 他のサービスやアプリとデータをやり取りするための仕組み・接続口。
- 例：Slack と Google カレンダーを連携し、予定を通知するなど。

2. CMS(Content Management System)

- Web サイトのコンテンツ（記事・画像など）を、専門知識がなくても管理画面から扱えるシステム。

3. プラグイン

- 基本機能に追加で機能を「プラス」できる拡張モジュールのようなもの。

4. 動的 Web コンテンツ

- ユーザーの操作（検索、支援、ログインなど）に応じて内容が変化する Web ページのこと。
